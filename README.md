# Dotfiles

A Nix flake managing every machine I touch — local desktops, an SBC, a fleet of cloud hosts (Oracle / Tencent / Huoshan), my macOS laptop, and the home-manager profile that follows me onto each of them.

![terminal](previews/terminal.png)

## Highlights

- **One flake, three platforms** — NixOS (`x86_64-linux` + `aarch64-linux`), nix-darwin (`aarch64-darwin`), and home-manager profiles shared across both.
- **Convention-driven** — [`ez-configs`](https://github.com/ehllie/ez-configs) auto-discovers hosts and modules from directory layout, so adding a new host is "drop a folder under `nixos-configurations/` and add it to `flake.nix:ezConfigs`".
- **Layered home-manager** — `cli-minimal → cli-server → cli-development` tiers let a tiny VPS share the same root profile as a full workstation without dragging in IDE-grade tooling.
- **Wayland-first desktop** — niri compositor with a Quickshell-based bar (noctalia), fcitx5 IME, GPU passthrough scripts for VFIO, and Sunshine/Moonlight for game streaming.
- **Encrypted secrets** — `sops-nix` with per-host age keys and PGP recipients (`.sops.yaml`).
- **Topology aware** — diagrams below are auto-generated by [`nix-topology`](https://github.com/oddlama/nix-topology).

## Quick Start

```bash
# Enter dev shell (sops, nixfmt, nixos-anywhere, statix, nixd, …)
direnv allow                # or: nix develop

# Build/deploy a NixOS host
nixos-rebuild switch --flake .#<hostname>
nixos-rebuild switch --flake .#<hostname> --target-host root@<ip>

# Build/deploy macOS
darwin-rebuild switch --flake .#office-mac

# Edit a SOPS-encrypted secret
sops secrets/common.yaml

# Build the installer ISO
nix build .#packages.x86_64-linux.iso
```

A more detailed working reference (host inventory, deploy targets, peripheral systems) lives in [`CLAUDE.md`](./CLAUDE.md).

## Using This Flake as an Upstream

You don't need to fork — the cleanest way to reuse these modules is to pull the flake in as an input and import only the pieces you want, the way [`autolife/nixos`](https://github.com/AutoLifeRobot) does internally. Your own repo only contains your hosts, your `shares.toml`, and your `secrets/` — every shared module stays in lockstep with upstream via `nix flake update`.

### 1. Bootstrap a thin flake

Create your own flake and add this repo as an input. **Mirror `inputs.follows` for every shared dependency** — without this you end up with two copies of nixpkgs in your closure and the overlays in `shared-modules/` may misbehave:

```nix
{
  inputs = {
    nixpkgs.url      = "github:NixOS/nixpkgs/nixpkgs-unstable";
    nixpkgs-stable.url = "github:NixOS/nixpkgs/nixos-24.11";
    flake-parts.url  = "github:hercules-ci/flake-parts";
    home-manager.url = "github:nix-community/home-manager";
    sops-nix.url     = "github:Mic92/sops-nix";
    srvos.url        = "github:nix-community/srvos";
    ez-configs.url   = "github:ehllie/ez-configs";
    nixos-hardware.url = "github:NixOS/nixos-hardware/master";
    # …whatever else upstream needs (niri, ghostty, lanzaboote, etc. — see this repo's flake.nix)

    dotfiles = {
      url = "github:xiongchenyu6/dotfiles";
      inputs = {
        nixpkgs.follows        = "nixpkgs";
        nixpkgs-stable.follows = "nixpkgs-stable";
        home-manager.follows   = "home-manager";
        sops-nix.follows       = "sops-nix";
        srvos.follows          = "srvos";
        ez-configs.follows     = "ez-configs";
        flake-parts.follows    = "flake-parts";
        nixos-hardware.follows = "nixos-hardware";
        # …mirror every shared input
      };
    };
  };
}
```

### 2. Provide your own `shares.toml` + `shares.nix`

The dotfiles modules read host/user metadata from `shares` (passed via `ez-configs.globalArgs`). Define your own — it's just TOML:

```toml
# shares.toml
[[users]]
gid = 1000
gn  = 'alice'
sn  = 'doe'
public-key = 'ssh-ed25519 AAAA…'

[[hosts]]
host = 'home-server'
network = 'lan'
public-key = 'ssh-ed25519 AAAA… root@home-server'
```

…with a thin loader:

```nix
# shares.nix
{ lib, ... }:
let shares = lib.importTOML ./shares.toml;
in {
  inherit (shares) users hosts;
  users-dict = builtins.listToAttrs (map (u: { name = "${u.gn}.${u.sn}"; value = u; }) shares.users);
  hosts-dict = builtins.listToAttrs (map (h: { name = h.host; value = h; }) shares.hosts);
}
```

### 3. Set up your own secrets

Each host gets an age key derived from its SSH host key — no manual key management needed:

```bash
# On each host you want to encrypt secrets *for*:
ssh root@<host> "cat /etc/ssh/ssh_host_ed25519_key.pub" \
  | nix shell nixpkgs#ssh-to-age -c ssh-to-age
# → age1qwerty… (paste this into .sops.yaml)
```

```yaml
# .sops.yaml
keys:
  - &me_pgp     0123ABCDEF…              # gpg --list-keys --keyid-format=long
  - &home_server age1qwerty…             # from ssh-to-age above

creation_rules:
  - path_regex: secrets/[^/]+\.yaml$
    key_groups:
      - age:
          - *home_server
        pgp:
          - *me_pgp
```

```bash
sops secrets/home-server.yaml          # edit; auto-encrypts on save
```

In your NixOS config, consume secrets through `sops-nix`:

```nix
sops.secrets."wireguard/private-key" = {
  sopsFile = ./secrets/home-server.yaml;
};
networking.wg-quick.interfaces.wg0.privateKeyFile =
  config.sops.secrets."wireguard/private-key".path;
```

### 4. Wire up `ez-configs` and import upstream modules

```nix
# flake.nix outputs
let
  shares = import ./shares.nix { inherit (nixpkgs) lib; };
in
flake-parts.lib.mkFlake { inherit inputs; } {
  imports = [ inputs.ez-configs.flakeModule ];
  systems = [ "x86_64-linux" "aarch64-linux" ];
  ezConfigs = {
    globalArgs = { inherit inputs shares; };
    root = ./.;
    nixos.hosts.home-server.userHomeModules = [ "root" "alice" ];
  };
};
```

Then in `nixos-configurations/home-server/default.nix`:

```nix
{ inputs, ezModules, ... }:
{
  imports = with inputs; [
    # Pick what you want from upstream à la carte:
    dotfiles.nixosModules.root            # root account + sshd hardening
    dotfiles.nixosModules.core            # sane Nix/locale defaults
    dotfiles.nixosModules.client-cli      # home-manager CLI tier (server-grade)
    dotfiles.nixosModules.gui             # desktop stack (skip on servers)
    dotfiles.nixosModules.wayland         # niri compositor + xdg portals

    # Your own host-specific modules live alongside:
    ./hardware-configuration.nix
    ezModules.my-extra-users
  ];

  # Per-host overrides on top of the imported defaults:
  networking.domain = "home.example.com";
  users.users.alice.extraGroups = [ "wheel" "docker" ];
}
```

Everything under [`nixos-modules/`](./nixos-modules), [`home-modules/`](./home-modules), and [`shared-modules/`](./shared-modules) is exposed automatically — see the directory listings to discover what's available.

### 5. Iterate

```bash
nix flake update dotfiles    # pull latest upstream (or pin a specific rev in flake.lock)
nixos-rebuild switch --flake .#home-server --target-host root@<ip>
```

Because `inputs.follows` is mirrored, upstream updates only change behavior of the modules you imported — your nixpkgs revision stays under your control.

## Layout

```
├── flake.nix                 Flake entry point — inputs, hosts, ez-configs mapping
├── shares.nix / shares.toml  Shared data (users, SSH keys, network config)
├── lib/                      Custom Nix library (Bird2/DN42 routing factory)
├── nixos-configurations/     Per-host NixOS system configs
├── nixos-modules/            Shared NixOS modules
├── darwin-configurations/    macOS system configs
├── darwin-modules/           Shared Darwin modules
├── home-configurations/      Per-user home-manager configs
├── home-modules/             Shared home-manager modules
│   ├── cli-{minimal,server,development}.nix    CLI tiers
│   ├── gui/                  Cross-platform + Linux-only GUI config
│   ├── niri/                 Niri compositor + spawn-at-startup wiring
│   └── nixos-integration.nix XDG portals, mime, user services
├── shared-modules/           Modules applied to both NixOS and Darwin (overlays, etc.)
├── secrets/                  sops-encrypted secrets
├── scripts/                  GPU passthrough, NixOS bootstrap, nixos-infect
├── stow-managed/             GNU Stow dotfiles (iTerm, IntelliJ, Rime, …)
└── ansible/                  Out-of-band playbooks (Nextcloud, Wazuh, sing-box, VersityGW)
```

## Module Tiers

```
cli-minimal        Base CLI tools — installed for every user (root included)
  └─ cli-server    Adds dev/ops tools (sops, ansible, k8s clients, …)
       └─ cli-development  Full IDE-like environment (LSPs, language toolchains)
```

Mix and match per home-config: a server profile imports up to `cli-server`; my workstation profile goes all the way to `cli-development` and adds `gui` + `niri`.

## Philosophy

- **Incremental progress** — small changes that compile and pass `nix flake check`.
- **Boring over clever** — clear intent beats clever expressions.
- **Composition over inheritance** — explicit module imports, no hidden state.
- **Single responsibility** — one concern per file; split when growth demands.
